== Overview

**iota-ssg** provides 28 static site generator (SSG) adapters that wrap various SSG CLIs and expose them through the Model Context Protocol (MCP). These adapters are synchronized from the link:https://github.com/hyperpolymath/poly-ssg-mcp[poly-ssg-mcp] hub.

=== What This Is

* A satellite project in the hyperpolymath ecosystem
* Collection of MCP adapters for 28 different static site generators
* Covers SSGs written in 16+ programming languages
* Follows Rhodium Standard Repository (RSR) guidelines

=== What This Is Not

* NOT the MCP server itself (that's link:https://github.com/hyperpolymath/poly-ssg-mcp[poly-ssg-mcp])
* NOT a replacement for the individual SSG tools

== Installation

These adapters are meant to be used with the poly-ssg-mcp server. Clone this repository to use the adapters directly:

[source,bash]
----
git clone https://github.com/hyperpolymath/iota-ssg.git
cd iota-ssg
----

== Usage

Each adapter exports a standard interface:

[source,javascript]
----
import * as zola from './adapters/zola.js';

// Connect to the SSG
await zola.connect();

// Use the tools
const result = await zola.tools[0].execute({ path: './my-site' });

// Disconnect
await zola.disconnect();
----

=== Adapter Interface

All adapters export:

* `name` - Display name of the SSG
* `language` - Implementation language
* `description` - Brief description
* `connect()` - Verify SSG is available
* `disconnect()` - Clean up resources
* `isConnected()` - Check connection status
* `tools` - Array of available MCP tools
